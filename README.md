# Stack Radar

**Stack Radar** — это инструмент для управления технологическим стеком, предназначенный для отслеживания использования технологий, языков программирования, фреймворков и инструментов в различных проектах организации.

Система обеспечивает прозрачность технологического ландшафта, позволяя командам эффективно управлять проектами, анализировать использование инструментов и принимать обоснованные технические решения.

## Возможности

*   **Технологический радар**: Централизованное отслеживание статуса технологий (например, Stable, Experimental, Deprecated).
*   **Панель управления**: Наглядная визуализация статистики использования технологий по категориям и проектам с помощью интерактивных графиков.
*   **Управление проектами**: Ведение реестра проектов с детальным описанием используемого технологического стека.
*   **Команды и сотрудники**: Организация структуры команд и распределение ролей участников.
*   **Безопасность**: Аутентификация пользователей и ролевая модель доступа (Администратор/Пользователь).

## Технологический стек

Проект построен с использованием современных технологий и практик:

### Backend
*   **Python 3.12**: Основной язык разработки.
*   **FastAPI**: Высокопроизводительный веб-фреймворк для построения API.
*   **AsyncPG**: Асинхронный драйвер для работы с PostgreSQL.
*   **Pydantic**: Валидация данных и управление настройками.
*   **JWT**: Безопасная аутентификация на основе токенов.

### Frontend
*   **React 18**: Библиотека для построения пользовательских интерфейсов.
*   **Vite**: Инструмент сборки и локальной разработки.
*   **TypeScript**: Типизированный JavaScript для надежности кода.
*   **TailwindCSS**: Утилитарный CSS-фреймворк для стилизации.
*   **Radix UI**: Набор доступных UI-компонентов.
*   **Recharts**: Библиотека для построения графиков.

### Инфраструктура
*   **Docker & Docker Compose**: Контейнеризация и оркестрация сервисов.
*   **PostgreSQL 16**: Реляционная база данных.
*   **Nginx**: Веб-сервер и обратный прокси.

## Начало работы

Для развертывания проекта необходимо наличие установленных [Docker](https://www.docker.com/) и [Docker Compose](https://docs.docker.com/compose/).

### Запуск приложения

1.  Необходимо клонировать репозиторий:
    ```bash
    git clone <repository-url>
    cd stack-radar-py
    ```

2.  Выполнить сборку и запуск контейнеров:
    ```bash
    docker-compose up -d --build
    ```

3.  После успешного запуска сервисов приложение будет доступно по следующим адресам:
    *   **Веб-интерфейс**: [http://localhost:8080](http://localhost:8080)
    *   **Документация API**: [http://localhost:8080/api/docs](http://localhost:8080/api/docs)

> **Примечание**: При первом запуске база данных автоматически инициализируется демонстрационными данными, включая пользователей, команды и технологии.

### Учетные данные

Для входа в систему можно использовать предварительно настроенные учетные записи:

| Роль | Email | Пароль |
|------|-------|--------|
| **Администратор** | `admin@local.com` | `password` |
| **Пользователь** | `user@local.com` | `password` |

## Конфигурация

Настройка приложения осуществляется через переменные окружения. Основные параметры определены в файле `docker-compose.yaml` и могут быть переопределены при необходимости.

Ключевые параметры конфигурации:

*   **База данных**: `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `POSTGRES_HOST`.
*   **Безопасность**: `SECRET_KEY` (используется для подписи JWT токенов, **требует изменения** в производственной среде), `ACCESS_TOKEN_EXPIRE_MINUTES`.
*   **Приложение**: `DEBUG` (режим отладки), `ALLOWED_ORIGINS` (настройка CORS).
